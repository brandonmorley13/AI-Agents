# harness — agent-harness methodology

The reusable methodology and building blocks behind these agents and skills.

## processes/
- `context-engineering.md` — how context is assembled and scoped for agent runs.
- `ensemble-review.md` — running multiple reviewers and reconciling their verdicts.
- `eval-harness.md` — evaluating agent/skill output quality.
- `incident-to-rule.md` — turning a caught bug/incident into a durable rule.

## templates/
- `Observable-Model.swift`, `SwiftUI-View.swift` — feature scaffolding templates.
- `Cloud-Function.js` — server function template.
- `Feature-Folder-README.md` — the shape of a feature module's README.

## examples/
- `CLAUDE.md` — an example project rule file the agents read as authoritative context.
